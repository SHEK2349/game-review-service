# GitHub Issues to Create

以下のイシューをGitHub CLIまたは手動でGitHubリポジトリに作成してください。

## Issue 1: プロジェクト初期セットアップとアーキテクチャ設計

**コマンド:**
```bash
gh issue create --title "プロジェクト初期セットアップとアーキテクチャ設計" --body "$(cat <<'EOF'
## 背景と目的

子供向けゲームレビューSNSの開発をスタートするため、プロジェクトの基盤となるアーキテクチャとセットアップを行う。音声入力によるレビュー作成とAI分析機能を持つスマホ最適化されたSNSプラットフォームの構築が目標。

## 実装内容

### 技術スタック設定
- React Native + Expo (フロントエンド)
- Node.js + Express + TypeScript (バックエンド)
- PostgreSQL + Redis (データベース)
- OpenAI API (音声処理・AI分析)
- Vercel + PlanetScale + AWS S3 (インフラ)

### プロジェクト構造
- モノレポ構成での初期セットアップ
- 基本的なディレクトリ構造の構築
- 開発環境とCI/CDパイプラインの設定

## 実装計画とタスク分解

### 実装順序（依存関係順）
- [ ] 1. Git リポジトリの初期化とブランチ戦略策定
- [ ] 2. モノレポ構成 (apps/mobile, apps/api, packages/shared)
- [ ] 3. React Native + Expo プロジェクト初期化
- [ ] 4. Node.js + Express + TypeScript API セットアップ
- [ ] 5. ESLint, Prettier, TypeScript設定
- [ ] 6. 環境変数管理設定 (.env, 各環境設定)
- [ ] 7. データベース接続設定 (Prisma ORM)
- [ ] 8. 基本的なAPI エンドポイントの作成
- [ ] 9. モバイルアプリの基本ナビゲーション設定
- [ ] 10. CI/CD パイプライン設定 (GitHub Actions)

### 並列実行可能なタスク
- フロントエンドとバックエンドの初期設定は並列可能
- ESLint/Prettier設定とGitHub Actions設定は並列可能

## 技術詳細

- **モノレポ管理**: npm workspaces
- **状態管理**: Zustand
- **API通信**: React Query
- **デザインシステム**: NativeBase
- **認証**: Firebase Auth

## 成功基準

- [ ] React Native アプリが iOS/Android で起動する
- [ ] API サーバーが起動し、基本エンドポイントが動作する
- [ ] データベース接続が正常に動作する
- [ ] TypeScript 型チェックが通る
- [ ] CI/CD パイプラインが動作する

## 制約事項

- 子供のプライバシー保護を最優先に考慮
- COPPA (Children's Online Privacy Protection Act) 準拠
- 日本の個人情報保護法への対応
- スマートフォンでの快適な動作 (60fps, 応答時間 < 300ms)
EOF
)" --label "enhancement,setup"
```

## Issue 2: ユーザー認証システムと保護者同意フローの実装

**コマンド:**
```bash
gh issue create --title "ユーザー認証システムと保護者同意フローの実装" --body "$(cat <<'EOF'
## 背景と目的

小学生〜高校生をターゲットとするため、年齢に応じた適切な認証フローと保護者同意システムが必要。COPPA準拠とプライバシー保護を最優先とした認証システムを構築する。

## 実装内容

### 認証機能
- 年齢確認機能（13歳未満は保護者同意必須）
- メール認証によるアカウント作成
- ソーシャルログイン（Google, Apple）
- 保護者同意メール送信・確認機能

### セキュリティ機能
- JWT ベースの認証
- リフレッシュトークン管理
- アカウントロック機能
- 不正アクセス検知

## 実装順序

- [ ] 1. Firebase Auth 設定とユーザーモデル設計
- [ ] 2. 年齢確認フォームの実装
- [ ] 3. 保護者同意メール機能の実装
- [ ] 4. JWT トークン管理の実装
- [ ] 5. ソーシャルログイン統合
- [ ] 6. セキュリティ機能の実装

## 成功基準

- [ ] 年齢に応じた適切な認証フローが動作する
- [ ] 保護者同意メールが正常に送信・確認される
- [ ] トークンの自動更新が動作する
- [ ] セキュリティテストをパスする

## 制約事項

- COPPA準拠必須
- 個人情報の最小限収集
- 適切なデータ暗号化
EOF
)" --label "feature,security"
```

## Issue 5: セキュリティとコンテンツフィルタリングの実装

**コマンド:**
```bash
gh issue create --title "セキュリティとコンテンツフィルタリングの実装" --body "$(cat <<'EOF'
## 背景と目的

子供向けSNSとして最重要なセキュリティとコンテンツフィルタリング機能を実装。健全なコミュニティ環境を維持し、不適切なコンテンツから子供を保護する。

## 実装内容

### 入口対策（投稿時フィルタリング）
- NGワードフィルター
- 不適切表現の建設的表現への誘導
- 投稿内容の事前チェック

### 出口対策（閲覧時保護）
- AIによる攻撃的・批判的コンテンツ検知
- 事前警告アイコン表示
- ぼかし機能（ワンタップで表示）

### 運用機能
- ユーザー通報機能
- 運営パトロール機能
- ガイドライン違反対応

## 実装順序

- [ ] 1. NGワードデータベースとフィルター実装
- [ ] 2. OpenAI Moderation API統合
- [ ] 3. コンテンツ警告システム実装
- [ ] 4. 通報機能の実装
- [ ] 5. 管理者ダッシュボード実装

## 成功基準

- [ ] 不適切なワードが適切にフィルタリングされる
- [ ] 問題のあるコンテンツに警告が表示される
- [ ] 通報機能が正常に動作する
- [ ] 管理者が効率的にコンテンツを監視できる
EOF
)" --label "security,moderation"
```

## インストール後の手順

1. GitHub CLIをインストール
2. GitHubにログイン: `gh auth login`
3. 上記コマンドを実行してイシューを作成
4. 各イシューに適切なラベルを設定
5. マイルストーンやプロジェクトに追加（必要に応じて）
